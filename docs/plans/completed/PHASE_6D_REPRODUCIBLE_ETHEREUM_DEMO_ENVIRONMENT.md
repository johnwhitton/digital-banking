## Minimal implementation design

- Add one root `compose.yaml`, one runtime-only `Dockerfile`, one deny-by-default `.dockerignore`, fixed public-address Anvil genesis/config material, and the prescribed `scripts/demo/` commands backed by one sourced POSIX shell library. Compose uses one named project/network, named PostgreSQL and Anvil volumes, loopback-only host ports, and the three approved digest-pinned images.
- Build the Spring Boot JAR with the committed Maven wrapper outside the image. The Temurin runtime image copies only the JAR and a small entrypoint, runs as a numeric non-root user, reads only named mounted secret/config files, and `exec`s Java.
- Preserve `.env.local-anvil` as the ignored ten-key authority map. Bootstrap generates only missing random PostgreSQL/sender/operator secrets under mode-`0700` `.demo-runtime/` with mode-`0600` files. Deployment metadata contains only chain ID, contract address, and verified public identities.
- Start Anvil without a committed mnemonic. A fixed local genesis funds only the ten already approved public development addresses. The application derives every configured signer address from the ignored keys; the deployer structurally validates the same fixed map, uses only Anvil's local auto-impersonated public identities, and receives no private value on a command line. Persist chain state in the named volume and load/dump its compressed RPC state token across ordinary stop/start.
- Use one replay-safe Foundry deployer script. It reads the existing locally compiled creation bytecode, deploys through the unlocked auto-impersonated owner address, grants ADMIN minter/burner roles, and verifies chain ID, bytecode, name, symbol, decimals, default admin, minter, burner, and the configured public address map. It writes only verified non-secret deployment data.
- Add one exact `local-demo-environment` profile. A smallest bearer filter reads the generated sender/operator token files, compares tokens in constant time, maps the sender to `local-demo/USER_1` with only Phase 6B/6C create/read authorities, maps the operator only to local-bank/demo-status read authorities, and fails closed on missing/equal/invalid token files. Default, test, `local-demo`, and `local-ethereum` alone remain credential-free.
- Add one read-only `GET /local/v1/demo/status` controller under the exact combined demo profile and a separate local OpenAPI resource. It joins fixed-fixture PostgreSQL evidence with read-only local-chain balance/supply calls and returns only exact bank/accounting/custody/supply positions, six bounded confirmed-effect counts, latest parent identifiers/statuses, payout-before-burn ordering, finality summaries, contract/network identity, and reconciliation. No mutation, key, token, raw transaction, unrestricted journal, internal reason, or arbitrary participant query is exposed.
- Demo A and B use only the documented REST APIs plus the operator status projection. Each begins from a clean known fixture, uses stable run-scoped idempotency keys, performs bounded polling, asserts every prescribed exact intermediate/final value and ordering/finality/effect count, and writes a redacted ignored JSON summary.
- Demonstrate restart recovery by accepting a parent, stopping/restarting only the control-plane container while PostgreSQL and Anvil persist, polling the same resource to a truthful terminal state, and comparing durable confirmed-effect counts to prove no duplicate. No failure-injection endpoint or mutable workflow control is added.
- `stop.sh` preserves all named state. `reset.sh --yes` resolves and removes only the exact Phase 6D Compose project volumes/network/containers and ignored Phase 6D runtime/output material; it never removes images, `.env.local-anvil`, repository data, Maven caches, or unrelated Docker resources.
